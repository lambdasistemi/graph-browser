# Data Model: Views

## Entities

### View File (`data/views/<name>.json`)

```json
{
  "name": "Cardano Governance",
  "description": "How on-chain governance works",
  "edges": [
    ["drep", "governance-action", "votes on"],
    ["cc-member", "governance-action", "ratifies"]
  ],
  "tours": [
    {
      "id": "basics",
      "title": "How Governance Works",
      "description": "Walk through the core concepts",
      "stops": [
        {
          "node": "drep",
          "depth": 1,
          "title": "Delegated Representatives",
          "narrative": "DReps vote on governance actions..."
        }
      ]
    }
  ]
}
```

### View Index (`data/views/index.json`, generated by build-action)

```json
[
  { "name": "Cardano Governance", "description": "...", "file": "governance.json" },
  { "name": "LLM Ecosystem", "description": "...", "file": "llm.json" }
]
```

## Relationships

- **Edge triple → Edge catalog**: Each `[source, target, label]` in a view references exactly one edge in `graph.json`
- **Tour stop → Node catalog**: Each stop's `node` field references a node in `graph.json`, but only nodes reachable from the view's edges are valid
- **View → Graph**: A view produces a filtered graph. Nodes = union of all sources and targets from selected edges. Edges = matched edges only.

## Backward Compatibility

| Repo layout | Behavior |
|-------------|----------|
| `data/` only, no `views/` | Full graph, tutorials from `data/tutorials/` |
| `data/` + `views/` | View picker available, "All" shows full graph with `data/tutorials/`, each view shows its subset with its tours |
| `views/` only, no `tutorials/` | View picker, "All" shows full graph with no tours |
